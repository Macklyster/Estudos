Caches

Os Caches em JPA, ajuda no desempenho do sistema, reduzindo os problemas de performance e o número de consultas ao banco de dados. Há diversas variedades que podem ser armazenadas em Cache na persistência, objetos, dados, conexões de bancos, declarações de bancos, entre outros. Veja um exemplo de anotação para Cache:

@Entity
@Cacheable
public class Carro {
...
}

Cache de Objeto

Os caches de objetos são armazenado nas entidades, a vantagem é que os dados são armazenados em cache no mesmo formato que é usado no Java. Eles são armazenados no nível do objeto e nenhuma conversão é necessária.

Data Cache

Os Data Cache armazena em cache os dados do objeto e não os próprios objetos. Os dados são normalmente uma representação da linha de banco de dados do objeto. A vantagem de um cache de dados é que é mais fácil de implementar, com isso você não tem que se preocupar com relacionamentos, identidade do objeto, ou gerenciamento de memória complexa.

Tipos de Cache

Existem muitos tipos diferentes de cache. O mais comum é um cache LRU, ou um que ejeta os objetos menos utilizado recentemente e mantém um número de tamanho fixo de MRU (mais recentemente usada) objetos. Alguns tipos de cache incluem:

  • LRU
  • Full
  • Soft
  • Weak
  • L1
  • L2
